# LubanVision 系统设计

## 1. 项目目标

构建一个可以通过SSH完成开发、部署和诊断的ROS 2主动视觉机器人。摄像头观察ArUco目标，
控制器根据目标中心与画面中心的误差调整云台，使目标持续保持在视野中心。

首版使用ArUco标记建立稳定、可测的视觉闭环。人体检测、YOLO和RK3588 NPU属于验收后
的增强功能，不得阻塞首版交付。

## 2. 闭环结构

```text
USB Camera
    │ sensor_msgs/Image
    ▼
aruco_detector ──→ /target/observation
                       │ center_x, center_y, visible, id
                       ▼
                 gimbal_controller
                 PID + 状态机 + 限位
                       │ /gimbal/command
                       ▼
                 soc_pwm_hardware
                       │ Linux PWM / PWM13_M1
                       ▼
                 SG90 pan / tilt
                       │ 改变摄像头方向
                       └──────────────→ Camera视觉反馈
```

SG90没有位置反馈，但视觉误差构成外环反馈。舵机目标角度和状态用于诊断，不能当作真实机械
角度测量。首版先完成单轴Pan，双轴Tilt复用同一接口。

## 3. 三机架构

- MacBook：Git主仓、项目文档、任务和问题管理、SSH运维。
- Windows/WSL2：ROS 2桌面开发、RViz、曲线观察、rosbag分析和离线测试。
- RK3588：Ubuntu 22.04 Server，运行相机、检测、控制、PWM硬件和systemd。

运行闭环不得依赖Mac或Windows在线。网络中断只影响远程观察，不影响RK本地控制；远程
命令超时后保持当前安全状态或回中。

## 4. ROS 2包规划

- `lubanvision_interfaces`：目标观测、云台状态以及启停/回中服务。
- `lubanvision_camera_cpp`：低开销V4L2相机输入、raw图像发布和速率探针。
- `lubanvision_vision`：Python视觉实验、ArUco检测、标注图像和检测统计。
- `lubanvision_control`：PID、目标丢失状态机、限位和模拟云台。
- `lubanvision_hardware`：RK3588 PWM访问、脉宽/角度标定和硬件诊断。
- `lubanvision_description`：`base_link`、`pan_link`、`tilt_link`、`camera_link`和TF。
- `lubanvision_bringup`：YAML参数、launch、rosbag配置与部署。

## 5. ROS 2接口

建议接口：

| 接口 | 类型 | 说明 |
|---|---|---|
| `/camera/image_raw` | `sensor_msgs/Image` | 原始图像 |
| `/target/observation` | 自定义消息 | ID、中心、归一化误差、可见状态 |
| `/target/debug_image` | `sensor_msgs/Image` | 带检测框调试图 |
| `/gimbal/command` | 自定义消息 | Pan/Tilt目标角度 |
| `/gimbal/state` | 自定义消息 | 命令角度、状态、PWM健康度 |
| `/tracking/enabled` | `std_msgs/Bool` | 显式启停跟踪 |
| `/tracking/reset` | `std_srvs/Trigger` | 清积分并回中 |

目标观测带相机时间戳；控制器忽略过期观测。参数包括目标ID、图像尺寸、PID、最大速度、
角度范围、丢失超时、回中时间和扫描范围，全部由YAML管理。

`lubanvision_interfaces/msg/TargetObservation`以源图像的`header`传递时间戳和坐标系；包含
目标ID、中心像素坐标、相对光心的像素误差和归一化误差、标记像素面积、置信度及检测状态。
像素坐标约定X向右、Y向下，归一化误差使用半幅宽/高归一化。只有`STATUS_DETECTED`默认可
驱动控制器；不可见时目标ID设为`-1`。OpenCV 4.5.4 ArUco不提供校准后的检测概率，首版
置信度采用二值语义：通过完整字典校验的目标为`1.0`，其余为`0.0`，不得解释为概率。
图像输入沿用M06验证的Reliable/Volatile QoS。调试图为640x480 BGR大消息，首版默认每15帧
发布一次，目标观测仍逐帧发布；性能统计中的P95包含调试图生成和序列化开销。

## 6. 控制策略

单轴初版：

```text
error_x = (target_x - image_center_x) / image_width
pan_delta = -(Kp*error_x + Ki*integral + Kd*derivative)
```

- `direction=-1`是首版默认方向；机械安装相反时只能显式改为`+1`，不允许隐式交换误差符号。
- 2026-07-21摄像头实际安装后用静止标记完成硬件A/B验证：`direction=+1`使绝对误差由
  0.590增至0.754，`direction=-1`使绝对误差由0.185降至0.035，因此该安装的正确方向明确
  固定为`-1`；不得再凭肉眼相对运动印象修改方向。
- 目标位于死区内时不更新角度，减少舵机抖动。
- 死区内不继续积分；`integral_limit=0`表示禁用积分，非零时进行对称抗饱和裁剪。
- 每周期限制最大角度增量，避免瞬间大动作。
- PID输出先做绝对值限幅，再相对上一周期输出做增量限幅；重置同时清积分、微分和增量历史。
- 输出经过机械角度限位和PWM限位双重裁剪。
- 目标丢失短于`hold_timeout`时保持；随后缓慢回中。
- 自动扫描是后续可选模式，默认关闭。
- 切换目标或重新捕获时清理PID积分，防止积分突变。

M14模拟闭环把PID输出解释为归一化Pan速度命令，模拟舵机使用一阶响应：

```text
actuator_rate += (command - actuator_rate) * dt / time_constant
error += actuator_rate * plant_gain * dt
```

验收模型使用`dt=0.05 s`、舵机时间常数`0.18 s`和单位增益。控制参数为
`Kp=1.5`、`Ki=0`、`Kd=0.12`、死区`0.003`、输出限幅`0.5`、周期增量限幅`0.08`。
这里的参数只用于验证软件方向、限幅和收敛语义，不等同于SG90硬件最终参数。首轮带积分参数
在死区附近形成低幅极限环，因此模拟验收保守关闭积分；M16-M17应基于真实舵机响应重新标定。

## 7. 硬件与电气

首版单轴不使用PCA9685，改用鲁班猫5物理Pin 32的`GPIO4_B6 / PWM13_M1`输出3.3V PWM信号。
SG90使用独立5V电源；舵机启动电流可能造成电压下降，因此5V支路应预留足够电流并在负载
附近增加电容。信号地、鲁班猫地和舵机电源负极必须共地，但舵机电流不得通过GPIO或3.3V
引脚供给。双轴阶段可使用另一硬件PWM，若通道或布线受限再评估PCA9685。

首次标定不假设通用的500～2500µs就是安全范围。先用保守脉宽和小角度移动，测出机械
边界，再把安全范围写入YAML。

`lubanvision_hardware`的sysfs PWM核心默认不提供自动扫描。M15-M16首轮只允许已经
冒烟验证的1.45-1.55ms临时范围；扩展标定后自动跟踪范围校正为围绕实测90度中心的
1.85-1.95ms，仍保持默认禁用。它拒绝接管已导出的通道，导出后先配置
周期、中位和极性，显式启用前再次恢复中位。应用异常、导出超时或写入失败时必须尝试禁用
和取消导出。

M17的`pan_tracking`节点继续保持默认禁用，仅在`/tracking/enabled`收到显式`true`后导出
PWM并从中位启动。节点只接受目标ID、检测状态、有限误差和源时间戳均有效的新鲜观测；控制
循环将PID输出解释为名义角速度，积分、输出、周期变化、名义角度和PWM脉宽逐层限幅。当前
`/gimbal/state`暂以JSON字符串发布启用状态、跟踪状态、名义角度、脉宽、新鲜度和故障，
M19建立正式自定义状态消息和bringup配置时替换该临时诊断接口。

M16首轮三点观察只确认1.45ms向左、1.50ms居中、1.55ms向右，当时的85/90/95度只是软件
名义值。后续手持短时扩展标定确认：1.00ms约0度、1.90ms约90度、2.50ms约150度；每个脉宽
都到固定位置后停止并可反向返回，因此该SG90是位置型而非360度连续旋转型。三个目测点符合
约10,000ns/度的线性关系，但最大角度不是180度，且仍未用量角器测量。

自动跟踪不直接使用端点范围。`ServoPwmConfig`和`ServoCalibration`默认只开放围绕实测90度
中心的1.85-1.95ms，对应名义85-95度；1.00-2.50ms只作为短时手持标定事实留档。供电升级、
机械固定复核、量角器复测和连续运行完成前，不得把扩展端点作为自动跟踪范围。

M17最终引导式闭环通过显式参数验证70-110度（1.70-2.10ms）测试边界、`Kp=6`、最大输出
5度/秒和周期变化0.75；实际该轮PWM为1.8368-1.9000ms，没有触及扩展边界。此配置记录在
平台YAML的`validated_live_test_profile`，表示可复现实验档案，不等于无条件替换保守默认。
方向必须保持`-1`；静止目标A/B证明`+1`使误差发散，而`-1`使误差约降低81%。

## 8. 故障与安全

- 相机超时：停止跟踪，保持短时间后回中。
- PWM导出、配置或写入失败：停止继续更新，发布故障，不循环高速重试。
- 检测节点退出：控制节点因观测超时进入安全状态。
- 参数非法：节点拒绝激活，例如最小角度大于最大角度。
- 系统启动：默认待命，必须显式启用跟踪。
- 节点退出：尽力发送中位命令，但安全不能只依赖析构函数。

执行任务，避免一次同时处理环境、视觉、控制和硬件。任务卡状态、顺序和验收产物以
[项目进度](03_progress_status.md)为准，执行与留痕规则以
[半小时执行手册](08_execution_playbook.md)为准。
